# 🤖 대시보드 자동 갱신 에이전트 지침

이 저장소는 AI·반도체 주식 동향 대시보드(`stock-dashboard.html`)입니다.
예약 클라우드 에이전트 2개가 이 파일의 데이터를 자동 갱신합니다.

- **매일(daily) 작업** — 아침 1회: ① 주가 ② 발굴주 ③ 급반등 후보
- **매시간(hourly) 작업** — 1시간마다: ④ 주요 뉴스

각자 자기 담당 객체만 수정한다. 수정 후 **반드시 `git pull --rebase origin main`으로 최신화한 뒤** 커밋·푸시한다. (두 에이전트가 같은 파일을 건드리므로 충돌 방지)

---

## 공통 규칙 (엄수)
- **가격은 추측 금지.** 웹에서 실제 최신 종가를 확인해 기입. 확인 불가하면 해당 항목은 그대로 두기.
- JS 객체의 **구조·중괄호·쉼표·따옴표 문법을 절대 깨지 말 것**(깨지면 대시보드가 안 열림). 수정 후 해당 블록을 다시 읽어 문법 검증.
- 한국어로 작성. 한국 종목 tkr/코드는 6자리 종목코드.
- 변경할 내용이 없으면 아무것도 하지 말 것(불필요한 커밋 금지).

---

## ① 주가 갱신 (daily) — `const STOCKS = [ ... ]`
파일 상단의 `STOCKS` 배열에 미국·한국 주요 종목이 있다. 각 항목의 **`price:`(현재가)와 `prevPct:`(전일대비 %)** 숫자만 최신 종가 기준으로 갱신한다. 그 외 필드(target, beta, flowScore, notes 등)는 건드리지 않는다.
- 미국(USD): 소수점 2자리. 한국(KRW): 정수(원).
- 대상 예: 미국 SPCX·NVDA·TSLA·AMD·AVGO·MU·INTC / 한국 삼성전자(005930)·SK하이닉스(000660)·한미반도체(042700)·한화에어로스페이스(012450).

## ② 발굴주 갱신 (daily) — `const HIDDEN`, `const HIDDEN_CHEAP`
- `HIDDEN`: 우량 숨은 발굴주(미국 6 + 한국 6). 기준: AI·반도체에서 기관 매집 초기 신호(13F 신규편입·저PER·낙폭 대비 수급개선) + 대중 인지도 낮음.
- `HIDDEN_CHEAP`: 저가 발굴주(미국 3 + 한국 3). 기준: 주가 체급 낮고 소외됐지만 상승 잠재력.
- 항목 형식(유지): `{name, tkr, sig, px(저가주만·'M/D 종가'), whyMe, signal, buzz(1~5), buzzTxt, risk}`
- 너무 유명한 대형주(NVDA·TSLA·삼성전자 등)는 발굴주에서 제외. 매일 최소 2~3개는 새 종목으로 교체해 신선도 유지.

## ③ 급반등 후보 갱신 (daily) — `const REBOUND`
- `REBOUND = { US:[...], KR:[...] }`. 각 시장 3~4개.
- 항목 형식(유지): `{tkr:'티커/종목명', score:0~100정수, d:'한두 문장 근거'}`
- 기준: 최근 낙폭 과대 + 기관 지지/촉매 + 단기 반등 모멘텀. score 높을수록 반등 가능성↑. 그날 동향에 맞게 종목·점수·설명 갱신.

## ④ 주요 뉴스 갱신 (hourly) — `const NEWS`
- `NEWS = { US:[...], KR:[...] }`. 각 시장 6~8개, 최신 위주(오래된 건 제거).
- 항목 형식(유지): `{tag:'종목/주제', cls:'up|down|new', h:'헤드라인', m:'날짜 · 출처 · 한줄부연'}`
  - cls: `up`=호재(초록), `down`=악재(빨강), `new`=중립/신규(보라)
- 웹에서 대시보드가 다루는 종목 및 AI·반도체 시장의 **그 시각 최신 뉴스**를 찾아 갱신. 헤드라인은 사실 기반·간결하게.

---

## 작업 절차
1. `git pull --rebase origin main` 로 최신 상태 확보
2. 담당 객체(daily=①②③ / hourly=④)를 웹 리서치 결과로 갱신
3. `stock-dashboard.html`을 다시 읽어 수정한 블록의 JS 문법 검증
4. 커밋 & 푸시 (daily 메시지: `일일 갱신 YYYY-MM-DD`, hourly 메시지: `뉴스 갱신 YYYY-MM-DD HH시`)
   - 푸시 거절 시 `git pull --rebase` 후 재푸시
