# 🤖 매일 발굴주 갱신 에이전트 지침

이 저장소는 AI·반도체 주식 동향 대시보드(`stock-dashboard.html`)입니다.
예약 클라우드 에이전트가 **매일 아침 발굴주(숨은 종목) 리스트를 재리서치해 갱신**합니다.

## 매일 해야 할 일

`stock-dashboard.html` 안의 두 데이터 객체를 그날의 리서치 결과로 업데이트하고 커밋한다:

1. **`const HIDDEN`** — 우량 숨은 발굴주 (미국 6 + 한국 6)
   - 기준: AI·반도체 테마에서 기관 매집 초기 신호(13F 신규편입·저PER·낙폭 대비 수급개선) + 대중 인지도 낮음
2. **`const HIDDEN_CHEAP`** — 저가 발굴주 (미국 3 + 한국 3)
   - 기준: 주가 체급이 낮고(미국 한 자릿수~십몇 달러, 한국 수만원 이하) 소외됐지만 상승 잠재력

## 각 종목 객체 형식 (반드시 이 필드 유지)
```js
{name:'종목명', tkr:'티커/코드', sig:'한줄 신호', px:'가격 (저가주만, 날짜 표기)',
 whyMe:'왜 골랐는지(1인칭, 구체적 근거)',
 signal:'공개 신호',
 buzz:1~5, buzzTxt:'커뮤니티 버즈·심리(어떤 키워드로 언급 급증하는지)',
 risk:'핵심 리스크'}
```

## 규칙 (중요)
- **가격(px)은 추측하지 말 것.** 웹에서 실제 최신 종가를 확인해 적고 `(M/D 종가)` 식으로 날짜 표기. 확인 불가 시 px 생략.
- 미국/한국 시장 균형 유지(HIDDEN 각 6, HIDDEN_CHEAP 각 3).
- 한국 종목 tkr는 6자리 종목코드(예 '042700').
- 너무 유명한 대형주(NVDA·TSLA·삼성전자 등)는 발굴주에서 제외 — "숨은" 종목 위주.
- 기존 종목과 겹쳐도 되지만, 매일 최소 2~3개는 새 종목으로 교체해 신선도 유지.
- 데이터 객체 **구조/문법을 깨지 말 것** (JS 문법 오류 시 대시보드가 깨짐). 수정 후 중괄호·쉼표 검증.
- 커밋 메시지: `발굴주 갱신 YYYY-MM-DD`

## 작업 절차
1. 웹 검색으로 오늘자 AI·반도체 + 저가 소외주 동향, 기관 매집/버즈 신호 리서치
2. `stock-dashboard.html`의 `HIDDEN`, `HIDDEN_CHEAP` 값 업데이트 (구조 유지)
3. 변경분 커밋 & 푸시
